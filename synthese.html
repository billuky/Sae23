Timéo Suau

J’ai été chargé de la réalisation du site web dynamique et du script de récupération des données via le broker MQTT. J’ai intégré les mesures dans la base de données et mis en place leur affichage dans le site sous forme de tableaux avec les statistiques (min, max, moyenne). J’ai aussi automatisé l’enregistrement des données grâce à crontab. Ce travail m’a permis de renforcer mes compétences en développement web (HTML/CSS/PHP), en scripts bash, et dans la manipulation de bases MySQL. J’ai particulièrement apprécié de relier la partie technique (capteurs, données) à une interface utilisateur fonctionnelle.

Germain Durand

Je me suis occupé de la base de données sous PhpMyAdmin. J’ai créé les différentes tables nécessaires au bon fonctionnement du site, en respectant les contraintes du cahier des charges. J’ai aussi mis en place un dashboard Grafana connecté à InfluxDB pour visualiser en temps réel les données des capteurs. Ce projet m’a permis de mieux comprendre le lien entre les bases de données relationnelles et les outils de visualisation, ainsi que l’importance d’un bon modèle de données dans un projet connecté.

Moad El Khatib

Mon rôle a été centré sur la gestion du projet. J’ai réalisé le diagramme de Gantt et mis en place l’organisation sur Trello afin que chaque membre sache ce qu’il avait à faire. Cela m’a permis d’expérimenter concrètement la gestion de tâches dans un projet en groupe. J’ai appris à planifier, répartir les rôles, suivre l’avancement, et à adapter notre rythme en fonction des imprévus techniques. Cette expérience m’a beaucoup aidé à comprendre l’importance d’une bonne organisation dans un projet informatique.

Florian Moine

Je me suis chargé de toute la partie Node-RED. J’ai mis en place les flux pour récupérer les données des capteurs via MQTT, les traiter, et les afficher dans un dashboard accessible sur une interface web. J’ai également travaillé sur l’intégration de ce dashboard dans le site global du projet. Ce travail m’a permis de découvrir la logique des flux Node-RED et l’importance de bien configurer les communications entre les différents services (broker, base de données, interface web).
